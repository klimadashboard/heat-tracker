# `data/` — runtime data files

The app server (`/api/current`, `/api/grid`) and the fetch pipeline
(`scripts/fetch-dwd.py`, run hourly via Coolify's scheduled task) read and
write this directory. On a deploy it **must be a persistent volume** so the
generated files survive container restarts.

## Committed to git (small, stable inputs)

| File | Purpose |
| --- | --- |
| `population-grid.json` | The fixed ~175k-cell grid (lat/lon/country/pop). Source of truth for cell order. |
| `nuts-by-cell.json` | NUTS-1/2/3 region name per cell, aligned to `population-grid.json`. Built by `scripts/build-nuts-grid.py`. Used by `fetch-dwd.py` to stamp `region` onto the grid GeoJSON. |

## NOT in git — must exist on the server volume

### `climatology-1961-1990.npz` — **required for the anomaly headlines**

The per-cell × day-of-year E-OBS 1961–1990 climatology, built by
`scripts/build-climatology.py` from the 4.3 GB `eobs-*.nc` source (also not in
git). It is ~404 MB, so it is deployed out-of-band rather than committed.

**If this file is absent, the pipeline still runs but writes `meanAnomalyC: null`
and `popAboveAvg: 0`** — i.e. the "+X °C hotter than average" and "uncommonly
hot" headlines have no data and are hidden in the UI. Only the "≥30 °C" headline
(which needs no reference period) shows. This is the single most common cause of
"the headlines are empty on staging/prod."

To deploy it, copy your locally-built `climatology-1961-1990.npz` into this
directory on the server volume (see the project root README / deploy notes for
the `rsync` + `docker cp` recipe), then trigger or wait for the next cron. The
array order **must** match the deployed `population-grid.json` (175,398 cells) —
`fetch-dwd.py` logs a cell-count mismatch and disables the climatology if not.

Check the cron log for:
- ✅ `Climatology loaded: climatology-1961-1990.npz (ref 1961-1990, …)`
- ❌ `Climatology file not found at … — anomaly/above-avg indicators will be NULL`
- ⚠️ `Climatology cell count (N) does not match population grid (M)`
